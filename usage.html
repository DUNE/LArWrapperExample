Usage¶

Installation¶

To use these scripts, first install this package from github, and metacat and data dispatcher from github (or ups).

Then run setup_gpvm.sh after modifying for your own user.

 1# setup for running metacat and data dispatcher tests
 2# assumes you have already done: 
 3#source /cvmfs/dune.opensciencegrid.org/products/dune/setup_dune.sh
 4export DATA_DISPATCHER_URL=https://metacat.fnal.gov:9443/dune/dd/data
 5export DATA_DISPATCHER_AUTH_URL=https://metacat.fnal.gov:8143/auth/dune
 6export METACAT_SERVER_URL=https://metacat.fnal.gov:9443/dune_meta_prod/app
 7export METACAT_AUTH_SERVER_URL=https://metacat.fnal.gov:8143/auth/dune
 8export PATH=$HOME/.local/bin/:$PATH
 9export HERE=$PWD
10export PYTHONPATH=$HERE/python:$HERE/scripts:${PYTHONPATH}
11export SQL_QUERY_PATH=$HERE/sql
12setup metacat
13# get some credentials
14kx509
15voms-proxy-init -rfc -noregen -voms=dune:/dune/Role=Analysis -valid 120:00
16voms-proxy-info
17export X509_USER_PROXY=/tmp/x509up_u$(id -u)  #
18metacat auth login -m x509 schellma
19metacat auth whoami

Data Delivery: Choose your delivery method¶

generated/LArWrapper and generated/DDInterface set up data delivery projects using different interfaces and run over multiple files

Running over large datasets requires a data access system capable of delivering data from a set of files to multiple processes. This is implemented via

  • a master project which creates and owns a list of files and keeps track

  • a data catalog (samweb or metacat) that knows about data characteristics

  • a data location catalog (samweb or rucio) that knows where a given file can be found

  • a data delivery method, either a direct copy or xroot streaming from the location

  • the process running on a worker node - it has an ID and is assigned files to process

The data dispatcher, wfs system and samweb all support the concept of projects and worker processe and act reasonably similarly.

This package currently supports batch and interactive runs of LArsoft using the new data dispatcher and interactive tests with the older samweb interface.

Note

wfs will be added soon.
